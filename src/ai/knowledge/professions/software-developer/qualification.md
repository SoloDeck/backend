# Software Development Qualification Guide

## Purpose

Defines profession-specific interpretation rules for qualifying software development inquiries. This guide supplements the Lead Qualification Framework and does not define scoring.

---

## 1. Domain Scope

### Supported Engagements

- Websites
- Web applications
- Mobile applications
- SaaS platforms
- Internal business systems
- Customer portals
- E-commerce systems
- Booking systems
- CRM / ERP modules
- REST APIs
- Backend services
- Workflow automation
- AI software features
- Third-party integrations

### Out-of-Scope Engagements

- Construction
- Civil engineering
- Architecture
- Manufacturing
- Hardware installation
- Network infrastructure
- Legal, accounting, or medical services

---

## 2. Intake Field Interpretation

### Project Type

Treat the selected project type as the primary deliverable. If the inquiry and selected project type conflict, flag the inconsistency for clarification.

| Project Type       | Typical Deliverable            |
|--------------------|--------------------------------|
| Website            | Business website               |
| Web Application    | Interactive web system         |
| Mobile Application | Native or cross-platform app   |
| API / Backend      | Backend service or integration |

---

### Business Goal

Treat the business goal as the primary indicator of project intent.

Interpret business goals as software capabilities rather than technical implementations.

| Business Goal         | Typical Capability  |
|-----------------------|---------------------|
| Sell products         | E-commerce          |
| Accept bookings       | Booking system      |
| Manage employees      | Internal management |
| Customer self-service | Customer portal     |

Multiple unrelated business goals may indicate multiple projects.

---

### Target Users
Use target users to determine intended system actors . Examples:

- Customers
- Employees
- Administrators
- Public users
- Business partners

Missing target users reduce confidence in scope interpretation.

---

### Platforms Needed
Use the selected platforms to determine deployment scope. Do not infer additional platforms unless explicitly stated.

---

### Core Features
Treat core features as the strongest indicator of project scope. Group related features into business capabilities rather than evaluating them independently.

| Features                | Capability             |
|-------------------------|------------------------|
| Login, Orders, Payments | Customer Portal        |
| Inventory, Orders       | Inventory Management   |
| Calendar, Appointments  | Booking System         |

Large groups of unrelated features may indicate multiple independent systems.

---

## 3. Qualification Rubric

### Strong Qualification Evidence

- Project type identified
- Business goal clearly stated
- Target users identified
- Platform identified
- Core features identified

---

### Requires Clarification

- Generic project type ("need software")
- Ambiguous business goal
- Missing target users
- Missing platform
- Undefined core features
- Conflicting structured fields and inquiry

---

### Scope Consistency

Evaluate whether structured fields describe a single coherent software project.

Examples:

| Situation                                     | Interpretation             |
|-----------------------------------------------|----------------------------|
| Booking + Customers + Mobile                  | Consistent                 |
| Website + ERP + POS + AI Chatbot              | Possible multiple projects |
| Website selected, inquiry requests native app | Inconsistent               |

---

### Feasibility Signals

Identify, but do not reject:

- Scope disproportionate to budget
- Unrealistic timeline
- Multiple independent systems
- Insufficient information for estimation

---

## 4. Mixed-Domain Guidance

Separate software development work from requests belonging to other professions.

| Request               | Qualification        |
|-----------------------|----------------------|
| Build an online store | Software Development |
| Design a logo         | Out of Scope         |
| Run Facebook Ads      | Out of Scope         |

Qualify only the software development portion and flag remaining requests for manual assessment.
