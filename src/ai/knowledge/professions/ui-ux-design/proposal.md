# UI/UX Design Proposal Guide

## 1. Purpose

Defines profession-specific guidance for generating UI/UX design proposals from qualified leads. This guide supplements the proposal generation framework and does not define proposal formatting.

---

## 2. Supported Project Types

Generate proposals for engagements including:

- Mobile application design
- Website design
- Dashboard design
- Desktop application design
- Design system creation
- UX improvement
- Product redesign
- Landing page design

---

## 3. Proposal Structure

A UI/UX proposal should generally include:

1. Project summary
2. Design scope
3. Deliverables
4. Timeline
5. Pricing
6. Assumptions
7. Exclusions

---

## 4. Scope Interpretation

### Converting Client Requirements into Scope

Convert qualified intake information into design scope.

| Intake Information | Proposal Scope |
|-------------------|----------------|
| Design type | Primary design engagement |
| Business goal | Design objective |
| Target users | Intended user group |
| Design scope | Nature of design work |
| Expected deliverables | Design outputs |

---

### Handling Missing Information

If important information is unavailable:

- State it as a clarification item.
- Do not invent screens, user flows, or features.
- Limit the proposal to confirmed design scope.

---

### Handling Assumptions

Distinguish between:

- Confirmed requirements
- Reasonable design assumptions
- Design recommendations requiring approval

Do not present assumptions as confirmed project requirements.

---

## 5. Deliverable Guidance

### Typical Deliverables

Depending on project scope, deliverables may include:

- User flows
- Wireframes
- High-fidelity UI mockups
- Interactive prototype
- Design system
- UX research findings
- Component library
- Design specifications

---

### Deliverable Wording

Describe deliverables as completed design artifacts.

Prefer:

- Interactive prototype
- Mobile application UI
- Design system
- User flow documentation

Avoid:

- Draw screens
- Make Figma
- Design some pages

---

### Optional Deliverables

Only include when supported by the inquiry:

- Design revisions
- Developer handoff
- Usability testing
- Accessibility review
- Design documentation

---

## 6. Timeline Guidance

### Common Project Phases

Typical phases include:

- Requirements discovery
- UX research (if applicable)
- Wireframing
- UI design
- Prototype
- Design review
- Final handoff

---

### Timeline Estimation Guidance

Estimate timelines according to:

- Number of screens
- Design complexity
- Research requirements
- Revision rounds
- Deliverables requested

Avoid precise schedules when project scope is incomplete.

---

### Dependency Considerations

Identify dependencies including:

- Client feedback
- Brand assets
- Existing product
- Content availability
- Revision approvals

---

## 7. Pricing Guidance

### Pricing Factors

Consider factors such as:

- Number of screens
- Design complexity
- UX research
- Prototype complexity
- Design system creation
- Revision requirements

---

### What Affects Estimates

Estimated pricing may increase due to:

- Large applications
- Multiple platforms
- Extensive user research
- Advanced prototyping
- Complex design systems

---

### When Not to Estimate

Avoid estimating when:

- Design scope is unclear.
- Required deliverables are undefined.
- Number of screens or workflows cannot be determined.

Recommend clarification instead.

---

## 8. Common Assumptions

### Safe Assumptions

Unless stated otherwise, proposals may reasonably assume:

- Standard responsive layouts where applicable
- Industry-standard UI conventions
- Modern design tools
- Digital deliverables

---

### Assumptions Requiring Confirmation

Do not assume:

- Number of screens
- Brand guidelines
- Existing design assets
- Design system requirements
- Accessibility standards
- Number of revision rounds
- Developer handoff requirements

These should be confirmed with the client.

---

## 9. Out-of-Scope Requests

Do not include services outside UI/UX design unless explicitly requested and supported.

Examples include:

- Software development
- Logo and brand identity design
- Marketing strategy
- Copywriting
- Photography
- Video production

When mixed-domain requests are identified, generate the proposal only for the UI/UX design portion and identify remaining requests separately.
