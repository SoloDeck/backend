# UI/UX Design Qualification Guide

## Purpose

Defines profession-specific interpretation rules for qualifying UI/UX design inquiries. This guide supplements the Lead Qualification Framework and does not define scoring.

---

## 1. Domain Scope

### Supported Engagements

- Website design
- Mobile app design
- Dashboard design
- Desktop application design
- Product redesign
- User interface improvements
- Design systems
- UX research
- Wireframing
- User flow design
- Interactive prototyping

### Out-of-Scope Engagements

- Software development
- Graphic branding
- Logo and marketing asset design
- Digital marketing
- Copywriting
- Photography and videography
- Architecture and industrial design

---

## 2. Intake Field Interpretation

### Design Type

Treat the selected design type as the primary deliverable. If the inquiry conflicts with the selected type, flag the inconsistency.

| Design Type | Typical Deliverable |
|-------------|---------------------|
| Website | Responsive website UI |
| Mobile App | Mobile application UI |
| Dashboard | Dashboard interface |
| Desktop Application | Desktop software UI |
| Other | Clarification required |

---

### Business Goal

Treat the business goal as the primary indicator of design intent.

Interpret goals as user experience problems rather than visual requirements.

| Business Goal | Typical Design Focus |
|---------------|----------------------|
| Increase sales | Conversion-focused UX |
| Improve usability | UX redesign |
| Modernize interface | UI redesign |
| Reduce user confusion | Information architecture |

Multiple unrelated business goals may indicate multiple design engagements.

---

### Target Users

Use the target users to determine the intended audience and design priorities.

Examples:

- Customers
- Employees
- Administrators
- Business clients
- General public

Missing target users reduce confidence in UX decisions.

---

### Design Scope

Treat the selected scope as the expected level of work.

| Scope | Interpretation |
|-------|----------------|
| New Design | End-to-end product design |
| Redesign | Improve existing product |
| Improve Existing Design | Partial enhancement |
| Design System Only | Reusable UI components |

If the inquiry requests work outside the selected scope, flag for clarification.

---

### Expected Deliverables

Treat selected deliverables as the strongest indicator of project scope.

| Deliverables | Typical Engagement |
|--------------|--------------------|
| Wireframes + User Flows | UX planning |
| High-Fidelity UI | Visual interface design |
| Prototype | Interactive validation |
| Design System | Component library |
| UX Research | Discovery and user analysis |

Large combinations of unrelated deliverables may indicate multiple phases rather than a single engagement.

---

## 3. Qualification Rubric

### Strong Qualification Evidence

- Design type identified
- Business goal clearly stated
- Target users identified
- Design scope identified
- Expected deliverables identified

---

### Requires Clarification

- Generic request ("need a design")
- Unclear business goal
- Missing target users
- Undefined design scope
- Missing deliverables
- Conflicting structured fields and inquiry

---

### Scope Consistency

Evaluate whether the structured fields describe a single coherent design engagement.

| Situation | Interpretation |
|-----------|----------------|
| Mobile App + New Design + Prototype | Consistent |
| Dashboard + Design System + High-Fidelity UI | Consistent |
| Website + Mobile App + Desktop App + UX Research | Possible multiple engagements |
| Design System Only selected but inquiry requests full product redesign | Inconsistent |

---

### Feasibility Signals

Identify, but do not reject:

- Large design scope with minimal budget
- Unrealistic delivery timeline
- Multiple independent products
- Insufficient information for estimating design effort

---

## 4. Mixed-Domain Guidance

Separate UI/UX design work from requests belonging to other professions.

| Request | Qualification |
|---------|---------------|
| Design a mobile banking app | UI/UX Design |
| Build the mobile app | Out of Scope |
| Design a company logo | Out of Scope |
| Create marketing banners | Out of Scope |

Qualify only the UI/UX design portion and flag remaining requests for manual assessment.
